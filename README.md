# Delliza Bakery Demo Store

کاتالوگ آنلاین دلیزا با ۵۴ محصول، آماده دیپلوی روی Vercel.

## ساختار

- `index.html`: صفحه اصلی سایت
- `assets/css/styles.css`: تمام استایل‌ها
- `assets/js/products.js`: دیتای ۵۴ محصول دلیزا
- `assets/js/app.js`: فیلتر، جستجو، مرتب‌سازی، modal و رندر محصولات
- `assets/images`: تصاویر SVG دمو برای محصولات

## اجرا روی سیستم

کافی است فایل `index.html` را در مرورگر باز کنید.

برای اجرای لوکال با سرور ساده:

```bash
python -m http.server 3000
```

بعد آدرس زیر را باز کنید:

```text
http://localhost:3000
```

## دیپلوی روی Vercel

1. پوشه پروژه را داخل GitHub آپلود کنید.
2. در Vercel گزینه New Project را بزنید.
3. همین ریپو را انتخاب کنید.
4. Build Command را خالی بگذارید.
5. Output Directory را خالی بگذارید.
6. Deploy را بزنید.

## تغییر محصولات

برای تغییر محصول، قیمت، عکس یا توضیح فقط فایل زیر را ویرایش کنید:

```text
assets/js/products.js
```

قیمت‌ها به همان فرم کوتاه کاتالوگ وارد شده‌اند؛ مثلاً `95` در سایت به شکل `۹۵ هزار تومان` نمایش داده می‌شود.

## نکته

این نسخه استاتیک است و محصول‌ها از فایل `assets/js/products.js` خوانده می‌شوند. برای توسعه بعدی، همین ساختار قابل تبدیل به پنل مدیریت یا نسخه Next.js است.


## تغییرات نسخه کمینه

در این نسخه نوار معرفی، آمارهای هیرو، ویترین امروز و مجله حذف شده‌اند تا صفحه خلوت‌تر باشد. ترتیب اصلی سایت: معرفی کوتاه، سفارش اختصاصی، محصولات و فوتر است.

## پنل مدیریت برای GitHub

فایل `admin.html` به پروژه اضافه شده است. این پنل بدون دیتابیس کار می‌کند و برای GitHub Pages/Vercel مناسب است.

روش استفاده:

1. سایت را باز کن و وارد `admin.html` شو.
2. محصول‌ها را اضافه، حذف یا ویرایش کن.
3. برای دیدن تغییرات روی همان مرورگر، دکمه «ذخیره پیش‌نمایش در مرورگر» را بزن و صفحه اصلی را refresh کن.
4. برای انتشار واقعی، دکمه «دانلود products.js» را بزن.
5. فایل دانلودشده را در GitHub جایگزین مسیر زیر کن:

```txt
assets/js/products.js
```

6. Commit بزن. اگر GitHub Pages فعال باشد، نسخه جدید بعد از build/deploy نمایش داده می‌شود.

نکته امنیتی: چون این پروژه استاتیک است، ورود/رمز واقعی ندارد. گذاشتن رمز داخل JavaScript امنیت واقعی ایجاد نمی‌کند. برای پنل واقعی باید بعداً بک‌اند، دیتابیس و احراز هویت اضافه شود.

## مسیر عکس محصول

برای محصول جدید، اول عکس را داخل GitHub در یکی از مسیرهای زیر آپلود کن، بعد مسیرش را در پنل وارد کن:

```txt
assets/images/menu-photos/name.jpg
assets/images/name.png
```

مثال:

```txt
assets/images/menu-photos/new-cookie.jpg
```
